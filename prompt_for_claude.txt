# Role & Mission
你是一个资深的、精通响应式布局、高保真UI还原以及数据大屏项目（Dashboard）组件化开发的 Vue3 + TypeScript 前端开发架构师。
你的任务是在当前目录下的 `@Huazhi0301` 文件夹中，从零开始搭建一个结构极其清晰、高度可配置、且包含复杂交互的可视化大屏工程。

# Core Objectives & Design Fidelity
1. **高保真还原**：请仔细且反复地审查 `@design_reference\**` 中的设计图。你构建的最终网页效果必须完全忠实于这些高保真设计图。在编写每一块代码前，先在脑海中与图片进行像素级的对比。
2. **切图精准应用**：所有的图片素材和图标都存放在 `@slices\**` 文件夹中。你必须通过代码，将每一个切图极其精准地应用到对应组件的设计图位置上，确保没有错位或拉伸。
3. **数据驱动与高可配置性**：工程必须是高度组件化的。页面上的数量、文字内容、图片路径、以及数据关联关系，都必须通过 JSON 配置文件（例如 `src/config/data.json`）来驱动，禁止将数据硬编码在组件中。

# Tech Stack
- Framework: Vue 3 (Composition API, `<script setup>` 语法)
- Language: TypeScript
- Build Tool: Vite
- Styling: 原生 CSS / SCSS 或 TailwindCSS（需熟练运用 毛玻璃 backdrop-filter 和 发光 box-shadow 等高级视觉效果）
- State Management & i18n: Vue 的响应式系统，配合自行实现的简易 i18n JSON 切换逻辑。

# Layout & Functional Requirements
网页整体分为上、中、下三大固定大小的块，具体要求如下（请严格固定容器和组件各自的尺寸，暂时不做数量相关的自适应大小）：

### 1. 顶部区域：公司分类一级与二级菜单
- **一级菜单**：包含“储备中数量”、“实施中数量”和“推广中数量”三个核心指标组件。
- **二级菜单（左右两侧展示）**：点击上述任意一级指标后，在页面的左右两侧展示对应的具体公司列表（参考设计图中的全展开状态）。
- **企业详情弹窗**：点击左右两侧的任意具体公司后，弹出企业详情页面（参考 `显示企业详情.png`）。弹窗自上而下分为三块：
  1. 公司简要信息
  2. 公司业务核心场景（含进度条/流程图表现）
  3. 公司业务涉及的核心技能列表。

### 2. 中心区域：技能分类体系
- **一级分类**：中心放置“柔性装配”、“柔性码垛”、“柔性质检”三个主底座组件（本身不可点击）。纯色渲染。
- **二级分类（细分技能）**：这三个场景各包含多个子组件（如“上下物料”、“定位对齐”），它们在视觉上呈现向上展开的二级菜单树形结构。
- **悬浮交互**：鼠标悬浮在任意细分技能节点上时，必须平滑浮现其详细介绍信息框。

### 3. 底部区域：公司列表悬浮框
- 对应顶部的三个一级分类，在页面最底部呈现为三个固定的信息悬浮框。
- 要求内部列表内容实现**可滚动（Scrollable）**。

### 4. 隐藏交互机制：跨区域发光连线 (Core Feature)
- **数据结构要求**：在 JSON 配置文件中，你需要将“核心技能”配置为具体公司的属性数组。
- **连线交互**：当鼠标 hover（悬停）主界面左右两侧展示的任意“具体公司”节点时，页面必须生成**发光的动态连线**，将该具体公司与其在中心区域所对应的“细分技能”节点精准连接起来。（提示：可以在最上层实现一个基于 SVG `<path>` 或 Canvas 的绝对定位连线层，计算节点的getBoundingClientRect坐标进行动态绘制和发光动画）。

### 5. UI 美术与动效规范
- **材质效果**：除了中心的技能分类底座采用纯色外，其余所有适合透明背景的组件，**必须**使用“毛玻璃”（Glassmorphism: `backdrop-filter: blur(...)`）配合半透明背景色和轻微的内/外发光氛围（`box-shadow`）。
- **微动效**：适宜的交互组件在 hover 时应有轻微的放大效果（`transform: scale(1.05)`）和亮度提升。保持所有的 transition 曲线平滑统一（如 `cubic-bezier(0.4, 0, 0.2, 1)`）。

### 6. 右上角：设置菜单与国际化
- 点击右上角的设置按钮，弹出一个设置菜单，包含：“语言切换”、“声音大小”、“夜间/日间模式”。
- **语言切换**：必须具备真实功能。准备 `zh.json` 和 `en.json` 语料库（英文内容可先自行翻译生成 mock 数据），切换时全站文字响应式变更。
- 其他选项（声音、主题）暂时只做架空 UI。

### 7. 数据 Mock 与填充
- 设计图中目前只有“上海天成精工有限公司”等少量真实信息，在分析图片后使用这些信息之外，你需要根据配置文件的结构，使用已有的信息进行合理的循环和重复填充（Mock），确保 UI 看上去是饱满的。后续用户只需要替换你写好的 JSON 文件即可。

### 8. 工程质量与注释
- 所有的代码必须包含**详细、专业且易懂的注释**，特别是关于坐标计算、连线逻辑和毛玻璃属性调整的地方。
- 必须编写一份详尽的 `README.md`，介绍工程目录结构、如何配置 JSON 数据、以及如何启动和构建项目。
